連猴子都能懂得Git入門指南

Git為分散式版本控制系統，是為了更好管理Linux內核而開發的。

Git的數據庫分為遠端數據庫和本地端數據庫。

如果遠端數據庫中有其他人的修改歷史，那麼把它完整地複製下來，就可以接著進行工作了。

若上次的push到執行下一次的push的期間，有其他人push更新了遠端數據庫，而你沒有更新你的本地端數據庫，那麼你的push會被拒絕。

如果遠端數據庫和本地端數據庫的同一個地方都發生了修改的情況下，因為Git不能自動判斷要導入那一個修改內容於是就會發生錯誤。

發生衝突的地方，Git會修改檔案的內容，所以衝突的地方需要手動修改。

Leo Editor Tutorial

Leo 編輯器是一個功能特殊的軟體套件，不僅能作為集思廣益用的大綱編輯，程式編輯架構器，電子書編輯器，也是絕佳的網站管理套件。

Leo 編輯器畫面包含: 大綱區(專案的大綱內容)、內文區(與大綱對應的內文)、紀錄區(顯示執行結果之紀錄)、緩衝指令區(直接下特殊執行指令用)、執行狀態區(顯示執行狀態)、圖標指令區(以按鍵或圖像顯示可執行指令)。

使用者若希望充分發揮 Leo 編輯器的所有功能， 仍然必須對 Python 程式語法有一定程度的了解,，幾乎所有 Leo 編輯器的操作都在執行 Python 程式，因此推廣 Leo 編輯器另外一個附加功能就是可以讓更多人了解 Python 程式語言的特性，進而透過 Python 語言高度的黏合性整合更多優秀的程式管理典範。
